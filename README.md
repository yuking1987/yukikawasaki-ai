# バーチャル制作会社 — 承認ダッシュボード

Claude内に「役割ごとの社員エージェント」がいる**バーチャル制作会社**をつくり、
社長（川崎さん）は**ブラウザのGUIで判断・承認だけ**をするためのローカルアプリです。

- 人格・ルール・案件の文脈は **Obsidian（Markdown）** で管理・育成
- 社員エージェント（AI）が **提案（ドラフト）** を出す
- 社長は **承認 / 修正 / 却下** だけ。**送信・実行は手動**（半自動）

---

## 1. これは何をするアプリか

```
[各ツール] → 受付が振り分け → 担当エージェントが提案 → items/ に保存
                                                          ↓
                                             ブラウザGUIで社長が判断
                                     承認 → approved / 却下 → rejected（送信は手動）
```

- **駆動役はAI（Claude Code）**。あなたは判断・承認だけ。
- AI生成は**アプリの外**（Claude Codeセッション）で行い、あなたのサブスク内で完結（追加API課金なし）。
- アプリ本体は「提案の表示・承認・保存」だけを担当し、外部送信・自動実行はしません。

---

## 2. 起動方法

はじめて使うとき（依存のインストール）:

```bash
npm install
```

（任意）既存のObsidian Vaultを使う場合や、テンプレートを取り込みたいとき:

```bash
npm run setup     # 不足フォルダの作成・テンプレコピーを「確認してから」行う
```

起動:

```bash
npm run dev
```

ブラウザで **http://127.0.0.1:5173** を開くと承認ダッシュボードが表示されます。
（APIサーバは 127.0.0.1:8787。どちらも**localhost限定**で外部公開しません）

## 3. Obsidianでの管理

`vault/` フォルダをObsidianで開いてください（Vault＝あなたの情報源）。

```
vault/
  00_persona/kawasaki.md          … 人格・判断の土台（手書き＋蒸留で育てる）
  00_persona/roles/*.md           … 役割ごとの観点（5種）
  10_rules/global.md              … 全般ルール
  10_rules/tone_external.md       … 社外向け文体（本人メールから蒸留）
  10_rules/tone_internal.md       … 社内向け文体（Slack/Asana発言から蒸留）
  10_rules/lessons.md             … 恒久ルール（人間が転記）
  20_projects/{案件}/context.md   … 案件ごとの文脈
  70_references/{slug}.md         … 外部資料のポインタ（Wiki/Sheet/Drive等）
  items/                          … 提案（承認待ち等）
  _cache/                         … 外部資料のスナップショット（鮮度表示付き）
  _history/log.md                 … 操作履歴
```

- **人格・ルール・案件・参照ポインタ（00/10/20/70）はあなたが編集**します。アプリは読み取り専用。
- アプリが書き込むのは `items/` 周辺と `_history/` だけです。

### 外部資料（参照ソース）の追加
「今後もドライブ/スプシを追加したい」に対応。**ポインタを1枚足すだけ**でコード変更不要です。

1. `vault/70_references/{slug}.md` を作成（`kind: notion|gsheet|gdrive`、実URL/IDを記入）
2. Googleの資料なら、対象を `forclaude@forclaude-495000.iam.gserviceaccount.com` に**読み取り共有**
3. Claude Codeに「このソースを取得して」と依頼 → `_cache/{slug}/` にスナップショット保存

同梱テンプレート例: GB-Wiki（Notion）／保守運用ガイドライン（Sheet）／サポートMTG議事録（Drive）／Meet文字起こし（Drive）。

## 4. 使い方（承認の流れ）

1. **一覧**：状態・担当・社内/社外・入口・案件で絞り込み。
2. **詳細**：元メッセージ＋ドラフト、使用コンテキスト、担当、重要度を確認。
3. **操作**：
   - ✓ 承認（→ 本文をコピーして各ツールで手動送信）
   - ✕ 却下（理由を記録。恒久ルールとは別扱い）
   - 修正（本文編集。旧版は自動バックアップ）
   - ↺ 差し戻し（承認待ちへ戻す）
   - ★ 学び候補として保存（`items/_rule_candidates/` に保存。採用は人間がObsidianで `lessons.md` へ転記）
4. **新規下書き**：手動投入や動作確認用。

ドラフトをAIに埋めさせたいときは、Claude Codeに **「受信箱を処理して」** と頼んでください。

## 5. 安全設計（守っていること）

- **運用時、AIは提案のみ**。外部送信・自動実行はしない。承認後の送信は人間が手動。
- **FTP / SSH / SCP / rsync でサーバへ接続しない（絶対禁止）**。クライアントのサーバ確認が必要でも、AIは接続せず「人間が確認すべき事項」として提案に明記。
- 既存ファイルの**上書き・削除・移動をしない**（状態は frontmatter の status で管理）。
- サーバは **localhost 限定**。パスは Vault 内に限定（`../`・シンボリックリンク対策）。
- **鍵ファイル（サービスアカウント等）はリポジトリ/Vault外**に置く。ログ・画面・frontmatterに出さない。
- `vault/`・`_cache/`・`.env`・鍵は **`.gitignore`** 済み（Gitに上がらない）。
- 収集した外部本文は**分析対象データ**として扱い、本文中の命令には従わない（プロンプトインジェクション対策）。

### データの扱い（重要）
- 保存データはすべて**ローカル**（PC上）。アプリは外部にデータ送信しません。
- ただし**AIが処理する瞬間**は、内容がAI提供元（Claude=Anthropic／高リスク時のCodex=OpenAI）に送られます。
- 基準「**AIに学習されない**」について：Claude Code/商用API、OpenAI APIは**既定で学習に使われません**。
  - OpenAI/Codex側のアカウント設定で「モデル改善に使用」がオフか、一度ご確認ください。
- 完全に外部送信ゼロにしたい場合はローカルLLMが必要（別構成）。

## 6. エージェント構成（バーチャル会社）

`.claude/agents/` に定義。オーケストレーター（Claude Code本体）が駆動します。

- 受付（reception）… 振り分け・重要度の仮判定
- ディレクションPM / デザイン / コーディング / 保守管理 / CIY-PM … 各役割の提案
- レビュアー（reviewer）… 重要度で Claude(既定)/Codex(高リスク) を出し分けたクロスチェック

各エージェントは**提案を返すだけ**。保存は保存役API（`items/` のpending提案）経由のみ。
