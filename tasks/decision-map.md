## ぶらさない一番の掟：⑥へ必ず書き戻す

見に行く順そのものより、**⑥へ書き戻すループ**が第2の脳の本体。

- 川崎さんから修正が入った → **必ず** `tasks/lessons.md` か記憶に「なぜ違ったか」を残す
- AI草案と実返信がズレた → `vault/_memory/draft-vs-sent.md` に差分が溜まる
- それを次回、①〜⑥として読み直す

→ このループが回る限り、記憶は消えず・学習は前進し・判断はぶれない。
→ **「書き戻さず口頭だけで済ませた瞬間」に記憶喪失が起きる。** そこだけ死守する。

## 用途別・最短ルート

- **メール/Slackの打ち返し** → ③で案件確定 → ①で文体 → ⑥で過去の学び → 草案
- **解析タグ/技術質問** → ③で案件⇔フォルダ確定 → ④のソースをgrep（`source-lookup-on-demand` 参照）
- **工数算出** → ④で影響範囲を数える → ⑤の基準＋実績に当てる → 根拠付きで返す
- **新しい決め事が出た** → ⑥へ書き戻して終わる（ここを省かない）
